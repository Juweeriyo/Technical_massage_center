Clear Space
Always maintain clear space around the logo equivalent to the height of the “CERAGEM” wordmark.
Do not place text, graphics, or other visual elements within this protected area.

6. Imagery Style
Photography Guidelines
Use clean, professional imagery that reflects wellness, therapy, and care.
Preferred Style:
• Soft lighting — avoid harsh shadows or overexposed highlights
• Calm and relaxing environments — spa-like, clean, modern interiors
• Professional staff — therapists in clean uniforms, professional posture
• Clean and modern settings — minimal clutter, neutral backgrounds with subtle warm tones
Image Treatment:
• Noheavy filters; maintain natural color balance
• Slight warm tone to align with the brand’s caring personality
• Avoid stock-photo clichés; prefer authentic, candid moments
7. Patient Search by Number
Feature Overview
The system allows users to quickly search for a patient using their unique Patient Number (e.g., CER
000123).
Search Field Specifications
• Label: “Search by Patient Number…”
• Placeholder: Search by Patient Number...
• Prefix Icon: Magnifying glass (Search icon), #999999
• Suffix Icon: Clear button (X icon) appears when input has value
• Example Format: CER-000123
• Width: Full width of container (responsive)
• Max Width: 480px on desktop
Search Behavior
1. User enters the patient number.
2. System fetches and displays matching patient in real-time (or after search click).
3. If no match is found, display a friendly message:
> “No patient found. Try a different patient number.”
4. Empty state shows a magnifying glass icon with the message centered.
Validation Rules
• Patient number is required for search.
• Patient number must be in a valid format (CER- prefix followed by digits).
• Results are shown in real-time (debounced at 300ms) or after explicit search click.

Appendix: Accessibility Checklist
• Color contrast ratios meet WCAG AA standards (minimum 4.5:1 for text)
• All interactive elements are keyboard accessible
• Focus states are clearly visible (red outline for inputs, buttons)
• Form labels are associated with inputs
• Error messages are descriptive and actionable
• Icons have appropriate aria-label or are accompanied by text
• Tables use proper <thead>, <th scope="col">, and <caption> where applicable
